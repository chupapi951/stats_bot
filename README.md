# Telegram Mini App — Учёт заказов и прибыли

Полноценный Telegram Mini App на **React (Vite) + Node.js (Express) + MongoDB** для
ведения заказов, учёта прибыли, еженедельных отчётов и автоматических советов.

```
stats_bot/
├── server/   # Node.js + Express API, бот, WebSocket
└── webapp/   # React Mini App (Vite)
```

## Возможности

- 📊 **Главный экран** — KPI за неделю, график прибыли по дням, ТОП-3 товара.
- 📦 **Заказы** — таблица, фильтры по статусу, поиск по № или названию, карточка заказа.
- ➕ **Добавление заказа** — модальное окно с автонумерацией (#101, #102…).
- 🔁 **Статусы** — `created → shipped → completed | returned` (с подтверждением удаления).
- 📈 **Отчёт за неделю** — сводка, прибыль/выручка/себестоимость, ТОП-3, лучший день, график.
- 💡 **Советы** — автоматическая аналитика (возвраты, маржа, доставка, динамика продаж).
- 🔔 **Уведомления** — в Telegram-чат при создании и смене статуса.
- 📨 **Бот-команды** — `/start`, `/week`, `/orders`, `/advice`, `/subscribe`.
- 🌓 **Тема Telegram** — `tg.themeParams` (тёмная/светлая).
- ⚡ **Real-time** — WebSocket-обновления списка заказов.
- 📱 **Mobile-first** — нижнее меню (4 вкладки), плавающая кнопка «Добавить», безопасные отступы.

## Установка

```bash
# 1. установить зависимости
npm run install:all

# 2. настроить переменные окружения
cp server/.env.example server/.env
# отредактируйте server/.env (см. ниже)

# 3. собрать фронтенд
npm run build:web

# 4. запустить бэкенд (Express + бот)
npm start
```

После запуска сервер слушает `http://localhost:3000`, а собранный Mini App доступен
на том же порту. В Telegram Mini App нужно открывать по **публичному HTTPS** URL.

### Режимы БД (`DB_MODE`)

| Значение   | Описание                                                                                 |
| ---------- | ---------------------------------------------------------------------------------------- |
| `mongo`    | Обычный MongoDB (`MONGO_URI`, `MONGO_DB`). Рекомендуется для продакшена.                 |
| `memory`   | In-process MongoDB через `mongodb-memory-server`. Если он не установлен — автопереход на `file`. |
| `file`     | Лёгкий JSON-файл в `server/data/`. Подходит для локальной разработки и небольших данных. |

> По умолчанию в `.env.example` стоит `DB_MODE=memory` — это даёт «нулевой» старт:
> просто клонируйте, установите зависимости, запустите — и всё работает.

## Конфигурация (`server/.env`)

| Переменная      | Назначение                                                                  |
| --------------- | --------------------------------------------------------------------------- |
| `BOT_TOKEN`     | Токен Telegram-бота (от `@BotFather`).                                      |
| `WEBAPP_URL`    | Публичный HTTPS-URL Mini App (например `https://bot.example.com`).          |
| `PORT`          | Порт Express-сервера (по умолчанию `3000`).                                 |
| `ALLOWED_USERS` | Список разрешённых Telegram user ID через запятую (пусто = все).             |
| `DB_MODE`       | `mongo` / `memory` / `file`.                                                |
| `MONGO_URI`     | URI MongoDB (для `DB_MODE=mongo`).                                          |
| `MONGO_DB`      | Имя базы (по умолчанию `stats_bot`).                                        |
| `TESTING`       | `1` — разрешить `x-tg-user-id` заголовок для отладки вне Telegram.          |

## API (REST, защищён `X-Tg-Init-Data`)

| Метод  | Путь                        | Описание                            |
| ------ | --------------------------- | ----------------------------------- |
| GET    | `/api/health`               | Проверка работоспособности.         |
| GET    | `/api/dashboard`            | Дашборд: KPI, графики, последние.   |
| GET    | `/api/orders`               | Список заказов (?status, ?search).  |
| POST   | `/api/orders`               | Создать заказ.                      |
| GET    | `/api/orders/:id`           | Получить заказ.                     |
| PATCH  | `/api/orders/:id`           | Изменить цену/название/категорию.   |
| POST   | `/api/orders/:id/status`    | `{ "status": "shipped" }`.          |
| DELETE | `/api/orders/:id`           | Удалить заказ.                      |
| GET    | `/api/report/week`          | Сводный отчёт за текущую неделю.    |
| GET    | `/api/advice`               | Список советов.                     |

## Бот

Команды:

- `/start` — открывает Mini App на главном экране.
- `/week` — открывает раздел «Отчёт за неделю».
- `/orders` — открывает список заказов.
- `/advice` — открывает раздел «Советы».
- `/subscribe` / `/unsubscribe` — еженедельный отчёт в чат (по воскресеньям в 20:00).

Бот отправляет в чат уведомления:
- `Заказ #101 создан`
- `Заказ #101 переведён в статус «Выкуплен». Прибыль: 1200 ₽`

## Деплой Mini App

1. Залейте бота на любой VPS (Node.js 18+).
2. Постройте фронтенд: `npm run build:web`.
3. Запустите бэкенд: `npm start` (или через `pm2`/`systemd`).
4. Сделайте `https://your-domain` доступным (например, через Cloudflare, Caddy или Nginx).
5. В `@BotFather` → `/setdomain` укажите домен, в `/newapp` — привяжите Mini App к боту и
   установите `WEBAPP_URL=https://your-domain` в `.env`.
6. В `WEBAPP_URL` укажите **тот же** URL, что прописан в `@BotFather`.

## Локальная разработка

### Вариант 1 — только фронтенд (встроенный мок, без backend)

Самый быстрый способ попробовать приложение локально: данные хранятся
в `localStorage` браузера, между вкладками синхронизируются через
`BroadcastChannel`. Никакой бот, сервер или MongoDB не нужны.

```bash
npm install --prefix webapp
npm run dev:web
```

Откройте **http://localhost:5173/** — приложение само определит, что
нет Telegram WebApp SDK, и активирует мок-режим. Сверху появится
оранжевый баннер «Локальный мок-режим» и кнопка **«Сбросить демо»**.

Явное включение / отключение:

```
http://localhost:5173/?mock=1   # принудительно мок
http://localhost:5173/?mock=0   # принудительно реальный backend
```

При первом запуске автоматически создаются 8 демо-заказов
(`#101 … #108`) с разными статусами.

### Вариант 2 — фронтенд + backend

```bash
# в одном терминале
npm run dev:web      # http://localhost:5173 (Vite, с прокси на 3000)

# в другом
DB_MODE=file TESTING=1 npm run dev:server
```

`VITE_MOCK`-переменные и `?mock=` позволяют переключаться на лету.
